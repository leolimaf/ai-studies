## Phase 3: User Story 1 — Gerenciar Tarefas (Criar, Editar, Excluir) (Priority: P1) 🎯 MVP

**Goal**: O usuário pode criar novas tarefas, editar tarefas existentes e excluí-las. A listagem mostra título, prioridade, status e data de conclusão. Estado vazio exibe mensagem orientativa.

**Independent Test**: Abrir `http://localhost:4200`, criar uma tarefa com todos os campos, confirmar que aparece na lista; editar o título, confirmar o novo título; excluir a tarefa, confirmar que desaparece. Tentar criar sem título — confirmar mensagem de erro.

### Implementation for User Story 1

- [X] T012 [P] [US1] Create `TaskManager/src/app/components/task-list/task-list.component.ts` as standalone component: recebe `@Input() tasks: Task[]`, emite `@Output() editTask` e `@Output() deleteTask`; exibe indicador visual para tarefas com `dueDate` anterior à data atual
- [X] T013 [P] [US1] Create `TaskManager/src/app/components/task-list/task-list.component.html` rendering table/list with columns: título, descrição, prioridade, status, data de conclusão; linhas com botões Editar e Excluir; bloco de estado vazio quando `tasks` está vazio
- [X] T014 [P] [US1] Create `TaskManager/src/app/components/task-form/task-form.component.ts` as standalone component: `ReactiveFormsModule` com `FormGroup` para campos `title` (required, no-whitespace validator), `description`, `dueDate` (required), `priority` (required), `status` (required, default `Pendente`); suporta modo criação e modo edição via `@Input() task: Task | null`; emite `@Output() saved` e `@Output() cancelled`
- [X] T015 [P] [US1] Create `TaskManager/src/app/components/task-form/task-form.component.html` with labeled inputs for all fields, `<select>` for priority and status bound to enum values, error message display for title validation, Save and Cancel buttons
- [X] T016 [US1] Update `TaskManager/src/app/app.component.ts` to: inject `TaskService`; load tasks on init via `loadTasks()`; manage `showForm` toggle and `selectedTask` state; wire `createTask`, `updateTask`, `deleteTask` calls to `TaskService`; request confirmation before delete
- [X] T017 [US1] Update `TaskManager/src/app/app.component.html` to render `<app-task-form>` (conditionally) and `<app-task-list>` with all event bindings

**Checkpoint**: US1 totalmente funcional e testável de forma independente — CRUD completo operando contra o json-server.

## Phase 4: User Story 2 — Filtrar Tarefas (Priority: P2)

**Goal**: O usuário pode filtrar a listagem por prioridade, status ou data de conclusão. Filtros são combináveis e podem ser limpos.

**Independent Test**: Com tarefas de diferentes prioridades e status no `db.json`, selecionar filtro "Alta" → confirmar que apenas tarefas Alta aparecem. Selecionar filtro "Pendente" junto → confirmar que apenas Alta+Pendente aparecem. Clicar "Limpar filtros" → confirmar que todas as tarefas retornam.

### Implementation for User Story 2

- [X] T018 [P] [US2] Create `TaskManager/src/app/components/task-filter/task-filter.component.ts` as standalone component: `FormGroup` com campos `priority`, `status`, `dueDate` (todos opcionais, default `null`); emite `@Output() filterChange: EventEmitter<TaskFilter>` a cada mudança de valor; botão "Limpar filtros" que reseta o form e emite `TaskFilter` com todos os campos `null`
- [X] T019 [P] [US2] Create `TaskManager/src/app/components/task-filter/task-filter.component.html` with `<select>` for priority (options: todas / Baixa / Média / Alta), `<select>` for status (todas / Pendente / Em andamento / Concluída), `<input type="date">` for dueDate, and "Limpar filtros" button
- [X] T020 [US2] Update `TaskManager/src/app/app.component.ts` to handle filterChange
- [X] T021 [US2] Update `TaskManager/src/app/app.component.html` to include `<app-task-filter>`

**Checkpoint**: US2 totalmente funcional — filtros isolados e combinados funcionando via query params do json-server; mensagem de lista vazia exibida quando nenhuma tarefa corresponde.

## Phase 5: User Story 3 — Atualizar Status Inline (Priority: P3)

**Goal**: O usuário pode alterar o status de uma tarefa diretamente na listagem sem abrir o formulário de edição.

**Independent Test**: Com uma tarefa "Pendente" visível na listagem, alterar o `<select>` de status na própria linha para "Em andamento" — confirmar que o status atualiza imediatamente na interface e é persistido no `db.json`.

### Implementation for User Story 3

- [X] T022 [US3] Update `TaskManager/src/app/components/task-list/task-list.component.ts` to add `@Output() statusChanged: EventEmitter<{id: number, status: Status}>` and `onStatusChange(task: Task, newStatus: Status)` method
- [X] T023 [US3] Update task-list.component.html with inline status select [(ngModel)]="task.status" (change)="onStatusChange(task, task.status)">` with all Status enum options; import `FormsModule` in component
- [X] T024 [US3] Update app.component.ts to handle statusChanged via patchStatus from `<app-task-list>` calling `TaskService.patchStatus(id, status)` and refreshing the task in the local list without full reload

**Checkpoint**: US3 totalmente funcional — status se atualiza na linha sem navegação; alteração persiste no `db.json`.
